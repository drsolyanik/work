```
# 1. Удаляем старую версию
sudo apt remove --purge -y nodejs npm
sudo apt autoremove --purge -y

# 2. Скачиваем свежий архив (или используем уже скачанный)
cd /home/admin
wget https://nodejs.org/dist/v24.14.1/node-v24.14.1-linux-x64.tar.xz

# 3. Распаковываем в /usr/local
sudo tar -xJvf node-v24.14.1-linux-x64.tar.xz -C /usr/local --strip-components=1

# 4. Сбрасываем кеш путей
hash -r

# 5. Проверяем
node -v   # v24.14.1
npm -v    # 11.11.0
```

Что делать, если команды node или npm не находятся после установки?
Убедитесь, что /usr/local/bin есть в переменной PATH:

bash
echo $PATH
Если нет – добавьте в ~/.bashrc:

bash
export PATH="/usr/local/bin:$PATH"
source ~/.bashrc
Проверьте права:

bash
ls -l /usr/local/bin/node
Файл должен существовать и быть исполняемым. Если нет – возможно, вы забыли sudo при распаковке.

Если у вас несколько версий (например, осталась от nvm), удалите их или переключитесь на нужную.

1. Где искать настройки релея (Исходящая почта)
Самое частое: Postfix настроен так, чтобы отправлять всю исходящую почту на KSMG.

Файл: /etc/postfix/main.cf

Что искать: Параметр relayhost.

Если там указан IP-адрес вашего старого KSMG, например: relayhost = [192.168.1.10], то ваш план с подменой IP сработает идеально.

Сложный вариант: Проверьте параметр transport_maps. Если он есть, загляните в файл, на который он указывает (например, /etc/postfix/transport). Там могут быть прописаны правила маршрутизации для конкретных доменов через IP KSMG.

2. Как Postfix доверяет KSMG (Входящая почта)
Чтобы почтовый сервер принимал письма от KSMG без лишних проверок (спам-фильтров, лимитов), его IP должен быть в «белом списке».

Файл: /etc/postfix/main.cf

Что искать: Параметр mynetworks.

Убедитесь, что там прописан IP-адрес KSMG. Если вы подмените IP на новом сервере, Postfix продолжит ему доверять «прозрачно».

3. Проверка специфических портов
Иногда KSMG и Postfix общаются не по стандартному 25 порту, чтобы избежать петель или для разделения потоков.

Файл: /etc/postfix/master.cf

Что искать: Строки, начинающиеся с портов (например, 10025, 10026).

Если вы видите там настройки smtpd, где в параметрах указан IP старого KSMG, это критическое место. После переезда проверьте, чтобы новый сервер слушал те же порты, если они используются.

4. Что проверить прямо сейчас (Команды)
Чтобы не гадать по конфигам, посмотрите, куда реально уходит почта прямо сейчас.

Посмотрите текущий relayhost одной командой:

Bash
postconf relayhost
Поищите IP старого KSMG во всех конфигах Postfix:

Bash
grep -r "IP_СТАРОГО_KSMG" /etc/postfix/
Это самый надежный способ найти «забытые» настройки.

Проверьте логи в реальном времени:
Отправьте любое письмо и посмотрите в /var/log/mail.log (или /var/log/maillog), на какой IP Postfix пытается его передать. Вы увидите строку вида: relay=192.168.x.x[192.168.x.x]:25.
